## ­ЪїЪ Vision

Every week, incredible products are built by Indian makers Рђћ but they don't get the launchpad they deserve. **BharatHunt** changes that.

A place where:
- **Makers** ship their products and get real feedback from a community that cares
- **Early adopters** discover the next big thing before everyone else
- **The Indian tech ecosystem** gets the spotlight it has always deserved

No gatekeepers. No editorial review. Just pure, community-driven discovery Рђћ ranked weekly, reset every Friday.

## ­Ъј» Features

### ­ЪЉц For Makers
- **Launch instantly**, **save as draft**, or **schedule** for the next weekly cycle
  - Draft mode: save your work privately, come back to edit later
  - Schedule: auto-publish on Friday when the community is most active
- Beautiful product pages with logo, hero image, gallery, and video embeds
- Real-time community feedback via upvotes and threaded comments
- Maker profiles showcasing everything you've built

### ­ЪЌ│№ИЈ For the Community
- **Weekly ranked feed** (Friday Рєњ Thursday) Рђћ every product gets a fair shot
- One upvote per product per user Рђћ your vote matters
- Category filters: Developer Tools, AI/ML, SaaS, Design, Productivity, and more
- Nested comments for genuine conversations

### ­Ъћњ Built Right
- **All auth handled server-side** Рђћ JWTs in memory, refresh tokens in HTTP-only cookies with rotation
- **OAuth (Google/GitHub)** and **magic link** authentication via Supabase Auth
- **PKCE flow** Рђћ no client secrets needed, immune to authorization code interception
- **Rate limited, validated, sanitized** Рђћ Zod validation at every boundary, input sanitization
- **Ownership enforced everywhere** Рђћ you own your content, period

## ­ЪЈЌ Architecture

### Package Layout

```
packages/
  shared/      Рєљ Shared types & constants (used by both frontend & backend)
  backend/     Рєљ Express API server
  frontend/    Рєљ Next.js application
```

### Deep Module Convention

Every package follows the **deep module** pattern: a lot of behaviour behind a small interface.

```
packages/<name>/
  index.ts      Рєљ Entry point (public). Import this from outside.
  lib/          Рєљ Implementation: hidden from outside, free to import each other.
  tests/        Рєљ Co-located tests + fixtures (a subfolder, so private).
```

**Import rules:**
1. Code outside a package may import only that package's entry points (root files)
2. A package's own files import each other freely
3. Tests through the entry points Рђћ never into `lib/` of another package
4. No dependency cycles between packages

### Backend Layers

```
routes/         Рєљ HTTP route definitions (thin Рђћ delegates to services)
  РћюРћђРћђ productRoutes.ts
  РћюРћђРћђ authRoutes.ts

  РћюРћђРћђ auth.ts                    JWT verification (required & optional)
  РћюРћђРћђ validate.ts                Zod schema validation
  РћюРћђРћђ errorHandler.ts            Centralized error handling
  РћюРћђРћђ rateLimiter.ts             Rate limiting per endpoint
  РћћРћђРћђ asyncHandler.ts            Async error wrapper

services/       Рєљ Business logic (no HTTP awareness)
  РћюРћђРћђ productService.ts          CRUD, voting, listing
  РћюРћђРћђ authService.ts             OAuth, magic link, JWT + refresh tokens
  РћюРћђРћђ cloudinaryService.ts       Signed upload URL generation
  РћћРћђРћђ categoryService.ts         Category listing
```

## ­Ъџђ Getting Started

### Prerequisites

- Node.js >= 20
- npm >= 10
- A Supabase project (for auth + database)
- A Cloudinary account (for image uploads)

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/atpaawej/bh.git
cd bh

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp packages/backend/.env.example packages/backend/.env
```

Required environment variables in `packages/backend/.env`:

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string (Supabase pooler) |
| `SUPABASE_URL` | Supabase project URL |
| `SUPABASE_ANON_KEY` | Supabase anonymous key |
| `SUPABASE_SERVICE_KEY` | Supabase service role key (admin operations) |
| `JWT_SECRET` | Secret for signing JWTs (min 32 chars) |
| `JWT_EXPIRES_IN` | Access token TTL (default: 15m) |
| `REFRESH_TOKEN_SECRET` | Secret for hashing refresh tokens (min 32 chars) |
| `CLOUDINARY_CLOUD_NAME` | Cloudinary cloud name |
| `CLOUDINARY_API_KEY` | Cloudinary API key |
| `CLOUDINARY_API_SECRET` | Cloudinary API secret |
| `FRONTEND_URL` | Frontend origin (for CORS) |

```bash
# 4. Set up the database
npx prisma db push        # Create tables from schema
npx prisma db seed         # Seed categories

# 5. (Optional) Set frontend env
echo "NEXT_PUBLIC_API_URL=http://localhost:4000/api" > packages/frontend/.env.local

# 6. Start development servers
npm run dev
# Runs backend on http://localhost:4000 and frontend on http://localhost:3000
```

### Available Commands

| Command | Description |
|---|---|
| `npm run dev` | Start both backend + frontend in dev mode |
| `npm run dev:backend` | Start backend only |
| `npm run dev:frontend` | Start frontend only |
| `npm run build` | Build all packages |
| `npm run lint` | Lint all packages + check import boundaries |
| `npm run test` | Run all tests |
| `npm run format` | Format code with Prettier |
| `npm run clean` | Clean all build artifacts |
